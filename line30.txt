Programação é difícil porque ninguém explica direito. Este capítulo vai ser LONGO de propósito. Leia com calma, faça pausas, e volte aqui sempre que precisar. O objetivo é que você saia daqui **entendendo de verdade** cada conceito, não decorando.\n\n---\n\n# 🧠 MEGA SEÇÃO 1 — VARIÁVEIS: A BASE DE TUDO\n\n## 1.1 — O Que É Uma Variável? (Explicação Profunda)\n\nAntes de qualquer código, vamos entender o conceito na vida real.\n\n### A Analogia do Armário\n\nImagina que o seu computador é um **armário gigante com milhões de gavetas vazias**. Cada gaveta tem um espaço diferente. Umas são pequenas (cabem um papel), outras são médias (cabem uma caixa de sapato), e algumas são enormes (cabem um cobertor).\n\nO problema é: **o computador é muito organizado, mas muito burro**. Ele não sabe o que você quer guardar se você não avisar antes. Ele precisa de 3 informações:\n\n1. **Que TIPO de coisa você vai guardar?** (Isso define o TAMANHO da gaveta)\n2. **Qual o NOME dessa gaveta?** (Para você achar ela depois)\n3. **O que vai DENTRO dela?** (O valor inicial)\n\n### O Diálogo Com o Computador\n\nQuando você escreve uma variável, é como se você tivesse esta conversa:\n\n**Você:** \"Ei, computador! Preciso de uma gaveta.\"\n**Computador:** \"Certo. Que tipo de coisa você vai guardar? Se for um número pequeno, eu pego uma gaveta pequena. Se for texto longo, eu pego uma grande.\"\n**Você:** \"Vou guardar um número inteiro (sem vírgula).\"\n**Computador:** \"Beleza. Vou reservar a gaveta do tipo `int`. Qual o nome que você vai dar pra ela?\"\n**Você:** \"Vou chamar de `idade`.\"\n**Computador:** \"Fechado. A gaveta `int idade` está pronta. O que tem dentro?\"\n**Você:** \"O número `25`.\"\n\n### Em código Java, esse diálogo vira UMA linha:\n```java\nint idade = 25;\n```\n\n### Dissecando Cada Pedaço Dessa Linha:\n\n| Pedaço   | O que é              | Analogia                              |\n|----------|----------------------|---------------------------------------|\n| `int`    | O **TIPO** da gaveta | \"Quero uma gaveta para números inteiros\" |\n| `idade`  | O **NOME** da gaveta | \"A etiqueta colada na frente\"          |\n| `=`      | O **COMANDO** \"coloca dentro\" | \"Abre a gaveta e bota isso lá\"   |\n| `25`     | O **VALOR** guardado | \"O conteúdo da gaveta\"                |\n| `;`      | O **PONTO FINAL**    | \"Fim da ordem. Obrigatório no Java.\"  |\n\n> 💡 **Sacou?** A variável é só isso: **uma gaveta na memória do computador com um nome, um tipo e um valor dentro.**\n\n---\n\n## 1.2 — Por Que Variáveis Existem? Qual a Lógica?\n\nImagine um programa de banco. Quando você abre o app do banco e vê \"Saldo: R$ 1.500,00\", aquele número **R$ 1.500** está guardado em algum lugar. Ele não aparece do nada. Ele está **armazenado em uma variável** no servidor do banco.\n\nSem variáveis, o programa não conseguiria:\n- **Lembrar** do seu nome quando você faz login\n- **Guardar** o seu saldo\n- **Calcular** quanto sobra depois de um pagamento\n- **Comparar** se você tem saldo suficiente para uma compra\n\n**Variáveis são a MEMÓRIA do programa.** Sem elas, o programa seria como uma pessoa com amnésia total — não guardaria nenhuma informação.\n\n---\n\n## 1.3 — Onde as Variáveis Moram? (RAM vs Disco)\n\nMuitos iniciantes se perguntam: \"Para onde vai a variável quando eu desligo o computador?\".\n\n1. **Memória RAM:** É onde as variáveis vivem. É uma memória **ultra rápida**, mas **volátil** (apaga quando desliga). Quando você roda o seu programa, as variáveis nascem na RAM. Quando você fecha o programa ou desliga o PC, a RAM é limpa e as variáveis **morrem**.\n2. **Disco Rígido (HD/SSD):** É onde ficam os arquivos permanentes (fotos, vídeos, arquivos .txt).\n\n**Por que isso importa?** Porque criar variáveis \"gasta\" RAM. Por isso o Java pede pra você escolher o \"tamanho da caixa\" (byte, int, long) com cuidado — pra não desperdiçar memória.\n\n---\n\n## 1.4 — Os 3 Ingredientes Obrigatórios de Toda Variável\n\nToda variável no Java tem exatamente **3 partes**. Se faltar uma, o Java \"briga\" com você.\n\n```java\n//  TIPO   NOME    =  VALOR;\n    int    idade   =  25;\n    double preco   =  49.90;\n    String nome    =  \"Ferick\";\n    boolean ativo  =  true;\n```\n\n1. **O TIPO (A Gaveta)**: Diz ao computador **o que** vai ser guardado.\n2. **O NOME (A Etiqueta)**: O apelido da caixa. Use `camelCase`: `idadeDoCachorro`.\n3. **O VALOR (O Conteúdo)**: O que você está guardando. Usamos `=` para dizer \"coloca isso lá dentro\".\n\n---\n\n# 🔥 MEGA SEÇÃO 2 — TODOS OS TIPOS DE VARIÁVEIS (EXPLICAÇÃO COMPLETA E PROFUNDA)\n\nAgora vem a parte mais importante. Vou explicar **cada tipo** como se você nunca tivesse visto programação na vida.\n\n---\n\n## 2.1 — `int` (Número Inteiro) — ⭐ O REI dos Tipos\n\n### O que é?\n`int` é o tipo que guarda **números inteiros** — ou seja, números **sem vírgula, sem ponto, sem frescura**. Pode ser positivo, negativo ou zero.\n\n### Para que serve?\nPara guardar qualquer coisa que se conta em números inteiros na vida real:\n- Idade de uma pessoa: `int idade = 25;`\n- Quantidade de produtos: `int estoque = 150;`\n- Nível de um personagem: `int nivel = 42;`\n- Pontuação de um jogo: `int pontos = 9800;`\n- Ano de nascimento: `int anoNascimento = 1999;`\n\n### Por que usar `int` e não outro tipo numérico?\nPorque `int` é o **tipo padrão** do Java para números inteiros. Quando o Java vê um número como `42` no código, ele automaticamente pensa: \"isso é um int\". É o mais equilibrado em termos de tamanho (4 bytes na RAM) e capacidade (até ~2 bilhões).\n\n### Capacidade do `int`:\n- **Mínimo:** -2.147.483.648 (menos dois bilhões)\n- **Máximo:** 2.147.483.647 (dois bilhões)\n\nIsso é mais que suficiente para 99% dos programas do mundo real.\n\n### Exemplo prático COMPLETO com explicação linha por linha:\n\n```java\npublic class ExemploInt {\n    public static void main(String[] args) {\n\n        // Criando variáveis do tipo int\n        int idadeDoFerick = 22;        // Guarda a idade (número inteiro, sem vírgula)\n        int anoAtual = 2026;           // Guarda o ano (número inteiro)\n        int saldoDeVidas = 3;          // Vidas restantes em um jogo\n\n        // Fazendo CONTAS com int (isso é o poder das variáveis!)\n        int anoNascimento = anoAtual - idadeDoFerick;\n        // O Java pega o valor de 'anoAtual' (2026), subtrai 'idadeDoFerick' (22) = 2004\n        // E GUARDA o resultado 2004 dentro da nova variável 'anoNascimento'\n\n        System.out.println(\"Ferick nasceu em: \" + anoNascimento); // Imprime: 2004\n\n        // Alterando o valor da variável (a gaveta é REUTILIZÁVEL!)\n        saldoDeVidas = saldoDeVidas - 1;  // Perdeu uma vida! Agora é 2\n        System.out.println(\"Vidas restantes: \" + saldoDeVidas); // Imprime: 2\n    }\n}\n```\n\n### ⚠️ O que o `int` NÃO faz:\n```java\nint preco = 19.99;  // ❌ ERRO! O int não aceita vírgula/ponto decimal!\nint preco = 19;     // ✅ CORRETO! Sem vírgula.\n```\n\n### 🎯 Erros comuns com `int`:\n1. **Tentar guardar decimal:** `int x = 3.14;` → ERRO\n2. **Ultrapassar o limite:** Se passar de 2 bilhões, o número \"dá a volta\" e fica negativo (overflow)\n3. **Divisão inteira:** `int resultado = 10 / 3;` → Resultado é `3`, NÃO `3.33`! O Java joga o resto fora.\n\n---\n\n## 2.2 — `double` (Número Decimal) — 💰 O Tipo do Dinheiro\n\n### O que é?\n`double` é o tipo que guarda **números com casas decimais** (com ponto). No Java, usamos **ponto** e não vírgula para decimais: `19.99` e não `19,99`.\n\n### Para que serve?\nPara guardar qualquer coisa que precisa de precisão decimal:\n- Preço de um produto: `double preco = 49.90;`\n- Nota de um aluno: `double nota = 8.75;`\n- Temperatura: `double temperatura = 36.5;`\n- Peso de uma pessoa: `double peso = 72.3;`\n- Porcentagem de desconto: `double desconto = 0.15;` (15%)\n\n### Por que `double` e não `float`?\nO `float` existe, mas o `double` tem o **dobro** da precisão (por isso o nome \"double\" = duplo). No mercado de trabalho, **sempre use `double`** para dinheiro e cálculos. O `float` é impreciso demais.\n\n### Capacidade do `double`:\n- Pode guardar números com até **15-17 dígitos** de precisão\n- Vai de números extremamente pequenos a extremamente grandes\n\n### Exemplo prático COMPLETO:\n\n```java\npublic class ExemploDouble {\n    public static void main(String[] args) {\n\n        // Situação real: Calculando o total de uma compra\n        double precoProduto = 49.90;\n        int quantidade = 3;\n        double desconto = 0.10;  // 10% de desconto (0.10 = 10%)\n\n        // Calculando o total\n        double totalBruto = precoProduto * quantidade;\n        // 49.90 * 3 = 149.70\n\n        double valorDoDesconto = totalBruto * desconto;\n        // 149.70 * 0.10 = 14.97\n\n        double totalFinal = totalBruto - valorDoDesconto;\n        // 149.70 - 14.97 = 134.73\n\n        System.out.println(\"Total bruto: R$ \" + totalBruto);\n        System.out.println(\"Desconto (10%): -R$ \" + valorDoDesconto);\n        System.out.println(\"Total a pagar: R$ \" + totalFinal);\n    }\n}\n```\n\n### 🎯 Erros comuns com `double`:\n1. **Usar vírgula em vez de ponto:** `double x = 3,14;` → ❌ ERRO! Use `3.14`\n2. **Esperar resultado exato:** `0.1 + 0.2` pode dar `0.30000000000000004` (isso é normal em computadores!)\n3. **Esquecer que int / int = int**: `double x = 10 / 3;` → resulta em `3.0` e não `3.33`! Correto: `double x = 10.0 / 3;`\n\n---\n\n## 2.3 — `boolean` (Verdadeiro ou Falso) — 🔘 O Interruptor\n\n### O que é?\n`boolean` é o tipo mais simples que existe. Ele só guarda **DOIS valores possíveis**: `true` (verdadeiro) ou `false` (falso). Nada mais. Não existe \"talvez\".\n\n### Para que serve?\nPara representar qualquer situação que é **sim ou não, ligado ou desligado, verdadeiro ou falso**:\n- O usuário está logado? `boolean estaLogado = true;`\n- A conta está bloqueada? `boolean contaBloqueada = false;`\n- O jogador está vivo? `boolean estaVivo = true;`\n- O aluno foi aprovado? `boolean aprovado = false;`\n\n### Por que o `boolean` existe? Qual a lógica?\nPorque TODA decisão no programa precisa de um true/false. Quando você escreve `if (idade >= 18)`, o Java avalia `idade >= 18` e transforma isso em um `boolean`: se a idade for 20, vira `true`. Se for 15, vira `false`. **O `boolean` é o motor que faz o `if/else` funcionar.**\n\n### Exemplo prático COMPLETO:\n\n```java\npublic class ExemploBoolean {\n    public static void main(String[] args) {\n\n        // Situação real: Sistema de login\n        String senhaCorreta = \"java123\";\n        String senhaTentativa = \"java123\";\n        boolean senhaConfere = senhaTentativa.equals(senhaCorreta); // true\n\n        boolean contaBloqueada = false;\n        boolean estaLogado = false;\n\n        // A lógica do sistema:\n        if (senhaConfere && !contaBloqueada) {\n            // Se a senha confere E a conta NÃO está bloqueada...\n            estaLogado = true;\n            System.out.println(\"Login realizado com sucesso!\");\n        } else {\n            System.out.println(\"Acesso negado.\");\n        }\n\n        System.out.println(\"Usuário está logado? \" + estaLogado); // true\n    }\n}\n```\n\n### 🎯 Erros comuns com `boolean`:\n1. **Escrever True ou TRUE:** Tem que ser minúsculo: `true` e `false`\n2. **Usar 0 e 1:** No Java, `boolean x = 1;` dá ❌ ERRO! Não é como outras linguagens. É `true` ou `false`, ponto.\n3. **Comparar de forma redundante:** `if (estaLogado == true)` funciona, mas é feio. Escreva simplesmente `if (estaLogado)`.\n\n---\n\n## 2.4 — `String` (Texto) — 📝 O Tipo do Texto\n\n### O que é?\n`String` guarda **texto** — qualquer sequência de letras, números, símbolos, espaços, emojis, o que você quiser, desde que esteja entre **aspas duplas** `\"\"`.\n\n### ATENÇÃO MÁXIMA: String NÃO é um tipo primitivo!\nNotou que `String` começa com letra **MAIÚSCULA**? Isso é porque String é uma **Classe** (um objeto), não um tipo primitivo como `int`, `double` e `boolean`. Por isso ela tem superpoderes (métodos) que os primitivos não têm.\n\n### Para que serve?\n- Nome de um usuário: `String nome = \"Ferick\";`\n- Email: `String email = \"ferick@gmail.com\";`\n- Mensagem de erro: `String erro = \"Senha incorreta!\";`\n- Endereço: `String endereco = \"Rua Java, 404\";`\n- CPF (sim, como texto!): `String cpf = \"123.456.789-00\";`\n\n### Por que CPF e CEP são `String` e não `int`?\nPorque **você nunca vai fazer conta com um CPF**. Você não vai somar dois CEPs. E o `int` não aceita pontos e traços. CPF `\"012.345.678-90\"` começa com zero — se fosse `int`, o zero da frente sumiria!\n\n### A Diferença entre `10` e `\"10\"` (ISSO É MUITO IMPORTANTE!)\n\n```java\nint numero = 10;        // Isso é um NÚMERO. O Java sabe fazer contas com ele.\nString texto = \"10\";    // Isso é um TEXTO. O Java trata como se fosse uma palavra.\n\n// Prova:\nSystem.out.println(numero + numero);  // 20 (somou os números)\nSystem.out.println(texto + texto);    // \"1010\" (COLOU os textos lado a lado!)\n```\n\nO `+` com números **soma**. O `+` com Strings **cola** (concatena). São coisas completamente diferentes!\n\n### Exemplo prático COMPLETO:\n\n```java\npublic class ExemploString {\n    public static void main(String[] args) {\n\n        String primeiroNome = \"Ferick\";\n        String sobrenome = \"Silva\";\n        int idade = 22;\n\n        // Concatenação: Colando textos com o +\n        String nomeCompleto = primeiroNome + \" \" + sobrenome;\n        // \"Ferick\" + \" \" + \"Silva\" = \"Ferick Silva\"\n\n        System.out.println(\"Nome: \" + nomeCompleto);\n        System.out.println(\"Idade: \" + idade + \" anos\");\n\n        // Superpoderes da String (métodos):\n        System.out.println(\"Tudo maiúsculo: \" + nomeCompleto.toUpperCase());\n        // Imprime: \"FERICK SILVA\"\n\n        System.out.println(\"Tudo minúsculo: \" + nomeCompleto.toLowerCase());\n        // Imprime: \"ferick silva\"\n\n        System.out.println(\"Quantas letras tem? \" + nomeCompleto.length());\n        // Imprime: 12 (conta espaços também!)\n\n        // Comparando Strings: USE .equals(), NUNCA use ==\n        String senha1 = \"java123\";\n        String senha2 = \"java123\";\n        System.out.println(\"Senhas iguais? \" + senha1.equals(senha2)); // true\n    }\n}\n```\n\n### 🎯 Erros comuns com `String`:\n1. **Escrever `string` minúsculo:** É `String` com S maiúsculo! `string nome = \"X\";` → ❌ ERRO\n2. **Usar aspas simples:** `String nome = 'Ferick';` → ❌ ERRO! String usa aspas DUPLAS `\"\"`\n3. **Comparar com `==`:** `if (nome == \"Ferick\")` → ERRADO! Use `if (nome.equals(\"Ferick\"))`\n\n---\n\n## 2.5 — Os Outros Tipos Primitivos (byte, short, long, float, char)\n\n### `byte` — O Menor de Todos\n```java\nbyte idadeBebe = 2;  // Vai de -128 a 127. Usa pouquíssima memória.\n```\n**Quando usar:** Quase nunca no dia a dia. Útil apenas quando memória é crítica (milhões de dados).\n\n### `short` — O Médio\n```java\nshort diasNoAno = 365;  // Vai de -32.768 a 32.767.\n```\n**Quando usar:** Raramente. O `int` é preferido em quase todas as situações.\n\n### `long` — O Gigante  \n```java\nlong populacaoMundial = 8000000000L;  // PRECISA do L no final!\n```\n**Quando usar:** Quando o número passa de 2 bilhões (população mundial, IDs de banco de dados enormes).\n**⚠️ Não esqueça o `L`** no final do número! Sem ele, o Java acha que é `int` e dá erro.\n\n### `float` — Decimal Menos Preciso\n```java\nfloat temperatura = 36.5f;  // PRECISA do f no final!\n```\n**Quando usar:** Quase nunca. Use `double`. O `float` perde precisão.\n\n### `char` — Uma Única Letra\n```java\nchar letra = 'A';       // Usa aspas SIMPLES! Uma letra só!\nchar simbolo = '@';     // Pode ser símbolo\nchar numero = '7';      // Isso é o CARACTERE 7, não o número 7!\n```\n**Quando usar:** Quando precisa de exatamente UM caractere. É raro — normalmente usamos `String`.\n\n---\n\n## 2.6 — Tabela Resumão: Quando Usar Cada Tipo\n\n| Tipo      | Guarda                  | Exemplo                        | Quando Usar                         |\n|-----------|-------------------------|--------------------------------|-------------------------------------|\n| `int`     | Número inteiro          | `int x = 42;`                 | Contar coisas, idades, quantidades  |\n| `double`  | Número com decimal      | `double x = 3.14;`            | Preços, notas, pesos, temperaturas  |\n| `boolean` | Verdadeiro ou Falso     | `boolean x = true;`           | Sim/Não, Ligado/Desligado           |\n| `String`  | Texto                   | `String x = \"Oi\";`            | Nomes, endereços, mensagens         |\n| `long`    | Inteiro gigante         | `long x = 99999999999L;`      | Números acima de 2 bilhões          |\n| `float`   | Decimal menos preciso   | `float x = 1.5f;`             | Raramente — prefira `double`        |\n| `byte`    | Inteiro mínimo          | `byte x = 100;`               | Raramente — prefira `int`           |\n| `short`   | Inteiro pequeno         | `short x = 30000;`            | Raramente — prefira `int`           |\n| `char`    | Uma letra só            | `char x = 'A';`               | Raramente — prefira `String`        |\n\n---\n\n# ⚡ MEGA SEÇÃO 3 — OPERADORES (O QUE FAZ OS NÚMEROS FUNCIONAREM)\n\n## 3.1 — Operadores Aritméticos (A Matemática do Java)\n\n### O que são?\nOperadores aritméticos são os **símbolos** que fazem o Java calcular coisas. São quase iguais à matemática da escola, mas com algumas diferenças.\n\n### Para que servem?\nSem operadores, as variáveis seriam inúteis. Imagina um app de calculadora que não sabe somar? Os operadores são o que dá AÇÃO às variáveis.\n\n| Operador | Nome          | Exemplo       | Resultado | O que faz                              |\n|----------|---------------|---------------|-----------|----------------------------------------|\n| `+`      | Adição        | `10 + 3`      | `13`      | Soma dois números                      |\n| `-`      | Subtração     | `10 - 3`      | `7`       | Subtrai o segundo do primeiro          |\n| `*`      | Multiplicação | `10 * 3`      | `30`      | Multiplica (NÃO use X, use asterisco!) |\n| `/`      | Divisão       | `10 / 3`      | `3`       | Divide (⚠️ inteiro / inteiro = inteiro!) |\n| `%`      | Módulo (Resto)| `10 % 3`      | `1`       | Dá o RESTO da divisão                  |\n\n### Exemplo COMPLETO que mostra TODOS:\n\n```java\npublic class OperadoresAritmeticos {\n    public static void main(String[] args) {\n\n        int a = 10;\n        int b = 3;\n\n        System.out.println(\"Soma:           \" + (a + b));  // 13\n        System.out.println(\"Subtração:      \" + (a - b));  // 7\n        System.out.println(\"Multiplicação:  \" + (a * b));  // 30\n        System.out.println(\"Divisão:        \" + (a / b));  // 3 (NÃO é 3.33!)\n        System.out.println(\"Resto (Módulo): \" + (a % b));  // 1\n\n        // 💡 DICA: Para a divisão dar decimal, use double:\n        double resultado = (double) a / b;\n        System.out.println(\"Divisão real:   \" + resultado); // 3.3333...\n\n        // 💡 DICA: O módulo serve para saber se um número é par ou ímpar:\n        int numero = 7;\n        System.out.println(numero + \" é par? \" + (numero % 2 == 0)); // false\n    }\n}\n```\n\n### Atalhos que Todo Programador Usa:\n```java\nint contador = 10;\ncontador = contador + 1;  // Jeito longo (funciona, mas ninguém faz assim)\ncontador += 1;            // Jeito médio (atalho de atribuição)\ncontador++;               // Jeito profissional (incremento: soma +1)\ncontador--;               // Decremento: tira -1\n\nint saldo = 100;\nsaldo += 50;   // saldo agora é 150 (mesmo que: saldo = saldo + 50)\nsaldo -= 30;   // saldo agora é 120\nsaldo *= 2;    // saldo agora é 240\n```\n\n---\n\n## 3.2 — Operadores Relacionais (As Comparações)\n\n### O que são?\nSão operadores que **comparam** dois valores e retornam `true` ou `false`. São o ingrediente principal do `if/else`.\n\n### Para que servem?\nPara perguntar ao computador coisas como: \"Esse número é maior que aquele?\" ou \"Esses dois valores são iguais?\"\n\n| Operador | Significado      | Exemplo     | Resultado |\n|----------|------------------|-------------|-----------|\n| `==`     | Igual a          | `5 == 5`    | `true`    |\n| `!=`     | Diferente de     | `5 != 3`    | `true`    |\n| `>`      | Maior que        | `5 > 3`     | `true`    |\n| `<`      | Menor que        | `5 < 3`     | `false`   |\n| `>=`     | Maior ou igual   | `5 >= 5`    | `true`    |\n| `<=`     | Menor ou igual   | `3 <= 5`    | `true`    |\n\n### ⚠️ CUIDADO GIGANTE: `=` vs `==`\n- **`=`** (um igual) → **ATRIBUIÇÃO**: \"Coloca esse valor dentro da variável\"\n- **`==`** (dois iguais) → **COMPARAÇÃO**: \"Esses dois valores são iguais?\"\n\n```java\nint x = 10;         // Atribuição: coloca 10 dentro de x\nboolean teste = (x == 10);  // Comparação: x é igual a 10? Sim! true\n```\n\n---\n\n## 3.3 — Operadores Lógicos (AND, OR, NOT)\n\n### O que são?\nSão operadores que **combinam** várias condições `boolean` em uma só.\n\n### Para que servem?\nNa vida real, as decisões são complexas. \"Para entrar no show eu preciso ter **18 anos E ter ingresso**.\" Você precisa de dois `boolean` verdadeiros ao mesmo tempo.\n\n| Operador | Nome  | Significado                                    | Exemplo                 |\n|----------|-------|------------------------------------------------|-------------------------|\n| `&&`     | AND   | **TODAS** as condições devem ser true           | `true && true` → `true` |\n| `\\|\\|`  | OR    | **PELO MENOS UMA** condição deve ser true       | `false \\|\\| true` → `true` |\n| `!`      | NOT   | **INVERTE**: true vira false, false vira true   | `!true` → `false`       |\n\n### Exemplo COMPLETO:\n\n```java\npublic class OperadoresLogicos {\n    public static void main(String[] args) {\n\n        int idade = 20;\n        boolean temIngresso = true;\n        boolean contaBloqueada = false;\n\n        // AND (&&): As DUAS precisam ser verdadeiras\n        boolean entraNoShow = (idade >= 18) && temIngresso;\n        System.out.println(\"Entra no show? \" + entraNoShow); // true\n\n        // OR (||): Basta UMA ser verdadeira\n        boolean temDesconto = (idade < 12) || (idade > 60);\n        System.out.println(\"Tem desconto? \" + temDesconto); // false (nem criança, nem idoso)\n\n        // NOT (!): Inverte o resultado\n        boolean podeLogar = !contaBloqueada;\n        System.out.println(\"Pode logar? \" + podeLogar); // true (conta NÃO está bloqueada)\n    }\n}\n```\n\n---\n\n# 🏗️ MEGA SEÇÃO 4 — ESTRUTURAS DE CONTROLE (O CÉREBRO DO PROGRAMA)\n\n## 4.1 — `if` / `else` (Se / Senão) — 🧠 A Tomada de Decisão\n\n### O que é?\n`if` é a estrutura que permite ao programa **tomar decisões**. Em português: \"**SE** tal coisa for verdadeira, **FAÇA** isso. **SENÃO**, faça aquilo.\"\n\n### Para que serve?\nSem `if/else`, o programa seria um trem sem freio — faria sempre a mesma coisa. Com ele, o programa ganha **inteligência**.\n\n### Sintaxe (Como escrever):\n```java\nif (condicao) {\n    // Se a condição for TRUE, esse bloco roda\n} else {\n    // Se a condição for FALSE, esse bloco roda\n}\n```\n\n### Exemplo COMPLETO:\n```java\npublic class ExemploIfElse {\n    public static void main(String[] args) {\n\n        double notaDoAluno = 6.5;\n\n        if (notaDoAluno >= 7.0) {\n            System.out.println(\"APROVADO! Parabéns!\");\n        } else {\n            System.out.println(\"REPROVADO. Estude mais!\");\n        }\n        // Como 6.5 < 7.0, a condição é FALSE, então imprime: \"REPROVADO. Estude mais!\"\n    }\n}\n```\n\n### `else if` (E se...?):\nQuando existem MAIS de dois caminhos:\n```java\npublic class FaixaEtaria {\n    public static void main(String[] args) {\n\n        int idade = 25;\n\n        if (idade < 12) {\n            System.out.println(\"Criança\");\n        } else if (idade < 18) {\n            System.out.println(\"Adolescente\");\n        } else if (idade < 60) {\n            System.out.println(\"Adulto\");         // ✅ Esse será impresso!\n        } else {\n            System.out.println(\"Idoso\");\n        }\n    }\n}\n```\n\n---\n\n## 4.2 — `switch` — 🔀 O Seletor de Opções\n\n### O que é?\nO `switch` é uma alternativa ao `if/else if/else if` quando você está comparando **UMA variável contra vários valores fixos**. É como um cardápio numerado.\n\n### Para que serve?\nEm vez de escrever 10 `else if`, você usa o `switch` que é mais limpo e organizado.\n\n### Exemplo COMPLETO:\n```java\npublic class ExemploSwitch {\n    public static void main(String[] args) {\n\n        int diaDaSemana = 3;\n\n        switch (diaDaSemana) {\n            case 1:\n                System.out.println(\"Domingo\");\n                break;   // OBRIGATÓRIO! Sem break ele continua caindo nos próximos cases!\n            case 2:\n                System.out.println(\"Segunda-feira\");\n                break;\n            case 3:\n                System.out.println(\"Terça-feira\");  // ✅ Esse vai ser impresso!\n                break;\n            case 4:\n                System.out.println(\"Quarta-feira\");\n                break;\n            default:\n                System.out.println(\"Dia inválido\");\n                break;\n        }\n    }\n}\n```\n\n### ⚠️ NUNCA esqueça do `break;`! Sem ele, o Java executa TODOS os cases abaixo do que bateu.\n\n---\n\n## 4.3 — `for` (Loop) — 🔁 O Repetidor Controlado\n\n### O que é?\n`for` é uma estrutura de **repetição** (loop). Ele executa um bloco de código **várias vezes**, e VOCÊ controla quantas vezes.\n\n### Para que serve?\nImagina que você precisa imprimir \"Olá\" 100 vezes. Você vai escrever 100 linhas de `println`? Não! Usa o `for`.\n\n### Anatomia do `for`:\n```java\nfor (int i = 0; i < 5; i++) {\n//   ^^^^^^^^^^  ^^^^^  ^^^\n//   INÍCIO      CONDIÇÃO  INCREMENTO\n//   \"começa     \"enquanto  \"depois de\n//    em 0\"       i < 5,     cada volta,\n//               continue\"  soma +1\"\n}\n```\n\n### Exemplo COMPLETO:\n```java\npublic class ExemploFor {\n    public static void main(String[] args) {\n\n        // Imprimindo de 1 a 5\n        for (int i = 1; i <= 5; i++) {\n            System.out.println(\"Volta número: \" + i);\n        }\n        // Imprime:\n        // Volta número: 1\n        // Volta número: 2\n        // Volta número: 3\n        // Volta número: 4\n        // Volta número: 5\n\n        // Exemplo real: Tabuada do 7\n        System.out.println(\"\\n--- Tabuada do 7 ---\");\n        for (int i = 1; i <= 10; i++) {\n            int resultado = 7 * i;\n            System.out.println(\"7 x \" + i + \" = \" + resultado);\n        }\n    }\n}\n```\n\n---\n\n## 4.4 — `while` (Loop) — 🔄 O Repetidor Condicional\n\n### O que é?\n`while` é outra estrutura de repetição. A diferença do `for` é que o `while` repete **enquanto uma condição for verdadeira**, sem saber necessariamente quantas vezes.\n\n### Para que serve?\nQuando você **não sabe quantas vezes** vai repetir. Exemplo: \"Continue pedindo a senha **enquanto** o usuário errar.\"\n\n### Exemplo COMPLETO:\n```java\npublic class ExemploWhile {\n    public static void main(String[] args) {\n\n        int tentativas = 0;\n        String senhaCorreta = \"java123\";\n        String tentativa = \"\";\n\n        // Simula tentativas de senha (máximo 3)\n        while (tentativas < 3) {\n            tentativa = \"senhaErrada\"; // Na prática viria do Scanner\n            tentativas++;\n            System.out.println(\"Tentativa \" + tentativas + \": senha incorreta!\");\n        }\n        System.out.println(\"Conta bloqueada após \" + tentativas + \" tentativas.\");\n\n        // Contando de trás pra frente\n        System.out.println(\"\\nContagem regressiva:\");\n        int contagem = 5;\n        while (contagem > 0) {\n            System.out.println(contagem + \"...\");\n            contagem--;\n        }\n        System.out.println(\"Lançar!\");\n    }\n}\n```\n\n### ⚠️ Loop INFINITO (o pesadelo do programador):\n```java\nwhile (true) {\n    // Isso NUNCA para! O programa trava!\n    // Sempre garanta que a condição vai virar false em algum momento.\n}\n```\n\n---\n\n# 🔐 MEGA SEÇÃO 5 — CONSTANTES (`final`): A CAIXA COM CADEADO\n\nÀs vezes você quer uma variável que **nunca mude**, como o valor de PI ou a taxa de imposto.\n\n```java\nfinal double IMPOSTO = 15.0;\nIMPOSTO = 20.0; // ❌ ERRO!
